⌨️字符串挪移(baidu)

发表于2024-10-18|更新于2024-10-19|前端面试代码手撕
|字数总计:4k|阅读时长:17分钟|阅读量:

题目描述

  1. 题目：给定一个长度为 nnn 且只包含小写字母的字符串 SSS（下标从 1 开始）。进行 nnn 次操作，第 iii 次操作将 S[i]S[i]S[i] 移动到字符串的末尾。请输出经过 nnn 次操作后的字符串。

    • 输入描述：在一行中输入一个由小写字母组成的字符串 SSS，其长度为 nnn（1≤n≤1061 \leq n \leq 10^61≤n≤106）。

    • 输出描述：在一行中输出经过操作后的字符串。

  2. 输入输出示例

    • 示例1：输入 paectc，输出 accept

    • 示例2：输入 abqde，输出 bdaeq

解题思想

常规想法 O(n^2)

基于对题目的分析，假设第 iii 次操作后的字符串记为 SiS_iSi​。那么第 i+1i + 1i+1 次操作后的字符串 Si+1S_{i+1}Si+1​ 可以表示为：

Si+1=Si[0:i]+Si[i+2:n]+Si[i+1]S_{i+1} = S_i[0:i] + S_i[i+2:n] + S_i[i+1] Si+1​=Si​[0:i]+Si​[i+2:n]+Si​[i+1]

其中，Si[0:i]S_i[0:i]Si​[0:i] 表示字符串 SiS_iSi​ 的前 iii 个字符，Si[i+2:n]S_i[i+2:n]Si​[i+2:n] 表示字符串 SiS_iSi​ 从 i+2i+2i+2 到 nnn 的字符，而 S[i+1]S[i+1]S[i+1] 是要移动到末尾的字符。由于题目要求进行 nnn 次操作，而每次操作都需要按照上述公式更新字符串，因此整体的时间复杂度为 O(n2)O(n^2)O(n2)。

注意：String.slice(k1, k2) 的时间复杂度为 O(k1−k2)O(k1-k2)O(k1−k2)，因此一次字符串更新的时间复杂度为 O(n)O(n)O(n)，其中 nnn 是字符串的长度。

以下是基于该常规想法的两个简单实现，

该实现的每次操作都基于当前字符串更新字符串。经过 nnn 次操作后，字符串将更新 nnn 次，第 nnn 次更新的字符串就是最终的结果。时间复杂度为 O(n2)O(n^2)O(n2)。

该实现将每次的字符串更新修改为索引映射的更新，减少了因字符串裁切和创建带来的性能消耗。时间复杂度仍然为 O(n2)O(n^2)O(n2)。

规律抽象：基于上述案例分析，我们可以抽象出用于解题的一般性规律。

对于字符串 SSS，假设其长度为 lengthlengthlength，同时定义以下变量：

  1. 根据 crIndexcrIndexcrIndex 和 lengthlengthlength 确定本轮要移动的所有字符索引：crIndex,crIndex+2,crIndex+4,…crIndex, crIndex + 2, crIndex + 4, \dotscrIndex,crIndex+2,crIndex+4,…（不超过 lengthlengthlength 的索引）。

时间复杂度分析

  • 外部循环次数：观察上述案例分析可知，对于长度为 nnn 的字符串，第一轮进行 ⌈n/2⌉\lceil n/2 \rceil⌈n/2⌉ 次操作，第二轮进行 ⌈n−⌈n/2⌉⌉\lceil n - \lceil n/2 \rceil \rceil⌈n−⌈n/2⌉⌉ 次操作，依次递减，最后一轮进行 111 次操作。总轮次就是 while 循环的次数，记总轮次为 xxx，则有：

    n⋅2−x<1⇒x>log⁡2nn \cdot 2^{-x} < 1 \Rightarrow x > \log_2 n n⋅2−x<1⇒x>log2​n

    因此，外部循环的时间复杂度为 O(log⁡n)O(\log n)O(logn)。

  • 内部操作次数：每轮操作涉及字符的移动，初始时的时间复杂度为 O(n)O(n)O(n)，但随着轮次的增加，移动的字符数量逐渐减少。因此，虽然最坏情况下的内部操作为 O(n)O(n)O(n)，但实际移动的元素会逐渐减少。

综合来看，外部循环的次数为 O(log⁡n)O(\log n)O(logn)，而每次循环中的操作为 O(n)O(n)O(n)，因此总体时间复杂度为 O(nlog⁡n)O(n \log n)O(nlogn)。然而，考虑到后续操作中的移动字符数量逐渐减少，实际的时间复杂度可能会更小。

单向链表 O(n)

在题目中，对字符串的操作描述为：第 iii 次将当前字符串的第 iii 个字符移动到字符串末尾。由于数组中元素移动的时间复杂度较高，因此我们可以考虑使用单向链表来优化操作。

通过对任意字符串的操作过程的分析，我们需要定义至少四个指针：head：指向链表的头节点；current：指向当前要移动到末尾的节点；prev：指向当前要移动节点的前一个节点；tail：指向链表的尾节点。

解题步骤

Step1. 初始化链表：

  • 使用给定字符串初始化一个单向链表，记为 linkedList。
  • 初始化时，head、prev 和 current 指向链表的第一个节点，而 tail 指向链表的最后一个节点。

Step2. 执行移动操作：

  • 在每次移动操作中，我们首先需要将 prev 指向 current 节点的下一个节点。这样可以方便我们处理 current 节点的移动。
  • 将 current 节点移动到链表的末尾：将 current 指向的节点添加到 tail 的后面，并更新 tail 指针。
  • 更新 current 为 prev 指向的节点的下一个节点，这样在下一轮中我们可以继续处理下一个要移动的节点。
  • 当 current 指向 tail 时，表示已经完成 nnn 次操作，循环终止。

时间复杂度：该分析流程与[找规律](#找规律 O(nlogn))类似。由于在循环内部的操作涉及到链表节点的删除和追加，而这些操作在单向链表中可以在常数时间内完成，因此它们的时间复杂度为 O(1)O(1)O(1)。因此，整个算法的总时间复杂度为 O(n)O(n)O(n)，其中 nnn 是链表中节点的数量。

感触：使用单向链表实现还是蛮复杂的！
